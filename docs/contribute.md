# Contributing to mcaddon.dev

## Ways to Contribute

You can help by:

- Fixing spelling or grammar mistakes
- Improve existing documentation
- Adding new documentation pages
- Providing code examples
- Updating outdated information
- Suggesting new guides or tutorials

## Before You Start

- Make sure your contribution fits the purpose of the site.
- Keep documentation clear, concise and beginner friendly whenever possible.

## Getting Started

1. Fork the repository
2. Clone the repository:

   ```bash
   git clone https://github.com/<owner>/<repo>.git
   cd <repo>
   ```

3. Install dependencies:

   ```bash
   npm install
   ```

4. Start the development server:

   ```bash
   npm run dev
   ```

5. Lint and build the site to check for errors.

   ```bash
   npm run lint
   npm run build
   ```

## Writing Guidelines

- Use clear descriptive headings.
- Keep paragraphs short.
- Explain concepts before showing code.
- Prefer complete working examples.
- Use proper markdown formatting.
- Include screenshots only when they improve understanding.
- Link to related pages instead of duplicating content.

## Code Examples

- Keep examples as small as possible.
- Make them copy-and-paste ready.
- Use consistent formatting.
- Explain any non-obvious behavior.
- Include both JavaScript and TypeScript examples when practical.
- Prefer TypeScript unless JavaScript is specifically being demonstrated.
- Use modern ECMAScript syntax.
- Focus each example on a single concept.

## AI-generated Content

AI tools may be used to assist with writing, but contributors are responsible for ensuring all content is accurate, up to date, and follows the style of mcaddon.dev. Always verify code examples before submitting them.

## Attribution

When documentation is based on or adapted from an official source (such as `https://docs.feed-the-beast.com`), include the source at the bottom of the page.

Example:

```md
<small>
Source: <a href="https://docs.feed-the-beast.com/docs/marketplace/Addons/storage-drawers/API/compacting_recipe" target="_blank">FTB Docs</a>
</small>
```
